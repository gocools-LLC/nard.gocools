# Nard v0.1.1 Release Notes (2026-03-05)

Patch release for `nard.gocools` focused on operational resilience and trust hardening.

## Highlights

- Improved CLI `node status` resilience with configurable retries/backoff and tighter default timeout behavior.
- Hardened heartbeat routing against clock skew with bounded timestamp handling and deterministic expiry behavior.
- Added revocation propagation coverage to validate compromised-key rejection and replacement-key acceptance paths.
- Enabled release workflow artifact upload for cross-platform binaries.
